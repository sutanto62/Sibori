- Use Conventional Commits
- Only use the following type (`feat`, `fix`, `refactor`, `chore`, `docs`, `test`, `perf`, `build`, `ci`).
- Keep commits focused and meaningful; prefer smaller, logically grouped edits.
- Branch names: `feature/<short-topic>`, `fix/<short-topic>`, `chore/<short-topic>`.
